IPsec Protocol Reference
AH (Protocol 51)
  • Authentication only
  • Integrity checking
  • No encryption
  • Cannot traverse NAT
ESP (Protocol 50)
  • Encryption + Authentication
  • Confidentiality
  • Integrity checking
  • Supports NAT-T
IKE (Port 500)
  • Key exchange protocol
  • IKEv1 has 2 phases
  • Phase 1: ISAKMP SA
  • Phase 2: IPsec SA
🧠 Memory Aid — AH vs ESP
AH = Authentication Header (no Hiding) | ESP = Encrypts Stuff Properly
AH provides auth only (51). ESP provides encryption AND auth (50). ESP is the one you want for full security.

💻 Lab Practicals

⚠️

These labs are based on the HCIA-Security V4.0 Lab Guide. Use Huawei eNSP simulator or real USG series firewalls for practice.

L1
Lab 1: Firewall Login & Basic Configuration
▼
🎯

Objective: Login to Huawei USG firewall via web UI and CLI, configure basic interfaces and security zones.

Step 1: Console Login
CLI — Initial Login
# Default credentials: admin / Admin@123
Username: admin
Password: Admin@123
# System will prompt password change on first login
USG6000> system-view
[USG6000]
Step 2: Configure Interfaces
CLI — Interface Configuration
[USG6000] interface GigabitEthernet 1/0/1
[USG6000-GigabitEthernet1/0/1] ip address 192.168.1.1 255.255.255.0
[USG6000-GigabitEthernet1/0/1] service-manage all permit
[USG6000-GigabitEthernet1/0/1] quit
Step 3: Add Interface to Security Zone
CLI — Security Zone Assignment
[USG6000] firewall zone trust
[USG6000-zone-trust] add interface GigabitEthernet 1/0/1
[USG6000-zone-trust] quit
# Verify
[USG6000] display zone

L2
Lab 2: Security Policy Configuration
▼
🎯

Objective: Configure security policies to allow/deny traffic between zones using 5-tuple matching.

CLI — Allow Trust → Untrust (Internet Access)
[USG6000] security-policy
[USG6000-policy-security] rule name trust-to-untrust
[USG6000-policy-security-rule-trust-to-untrust] source-zone trust
[USG6000-policy-security-rule-trust-to-untrust] destination-zone untrust
[USG6000-policy-security-rule-trust-to-untrust] action permit
[USG6000-policy-security-rule-trust-to-untrust] quit
# Verify policies
[USG6000] display security-policy rule all
💡 Remember: Default action is deny. Traffic that doesn't match any rule is dropped. Always explicitly allow traffic you want to pass.
L3
Lab 3: Source NAT (Easy IP)
▼
🎯

Objective: Configure Easy IP (NAPT using outbound interface IP) for internal hosts to access the Internet.

CLI — Easy IP NAT Configuration
[USG6000] nat-policy
[USG6000-policy-nat] rule name easy-ip-rule
[USG6000-policy-nat-rule-easy-ip-rule] source-zone trust
[USG6000-policy-nat-rule-easy-ip-rule] egress-interface GigabitEthernet 1/0/3
[USG6000-policy-nat-rule-easy-ip-rule] action source-nat easy-ip
[USG6000-policy-nat-rule-easy-ip-rule] quit
# Verify NAT sessions
[USG6000] display nat session all
L4
Lab 4: IPsec VPN Site-to-Site
▼
🎯

Objective: Configure IPsec VPN tunnel between two sites using IKEv1/IKEv2.

CLI — IKE Proposal
[FW-HQ] ike proposal 10
[FW-HQ-ike-proposal-10] authentication-method pre-share
[FW-HQ-ike-proposal-10] encryption-algorithm aes-256
[FW-HQ-ike-proposal-10] dh group14
[FW-HQ-ike-proposal-10] quit
# IKE Peer
[FW-HQ] ike peer branch
[FW-HQ-ike-peer-branch] pre-shared-key cipher Huawei@123
[FW-HQ-ike-peer-branch] remote-address 200.1.2.3
[FW-HQ-ike-peer-branch] quit

🎯 Exam Tips & Strategy

H12-711 Exam Quick Facts

⚡ High-Frequency Exam Topics
  • 🔒 Firewall security zone priorities (Local=100, Trust=85, DMZ=50, Untrust=5)
  • 🌐 NAT types: NAT No-PAT, NAPT, Easy IP, Bidirectional NAT
  • 🔐 AH (protocol 51) vs ESP (protocol 50)
  • 🔑 Symmetric vs Asymmetric encryption differences
  • 👤 AAA: Authentication, Authorization, Accounting steps
  • 🔄 VRRP master election and VGMP groups
  • 🌍 VPN types and which layer they operate on
  • 🏛️ PKI components: CA, RA, End Entity
  • 💥 DDoS attack classification: Traffic attacks
  • 🔢 Hash algorithms: MD5 (128-bit), SHA-1 (160-bit)
🧠 Common Traps to Avoid
⚠️

AH Protocol Number: AH = 51, ESP = 50. Many students swap these!

⚠️

SSL VPN: Does NOT require a client (it's clientless). C is wrong for Q9.

⚠️

Caesar Cipher uses character substitution, NOT a stick/scytale. Scytale = ancient Greek transposition cipher.

⚠️

NAT Server server-map table does NOT include Source IP — only Destination IP, Destination port, Protocol.

⚠️

RADIUS auth uses UDP, HWTACACS uses TCP. Port 1812 = RADIUS auth, 1813 = RADIUS accounting.
